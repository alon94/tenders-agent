## ✅ TICKET-11 — אי-עקביות תאריכים בין הרשימה לדף הפרט + תאריך פרסום ריק
**סוג:** Bug · **עדיפות:** גבוהה · **סטטוס: בוצע (יולי 2026)**

- נוסף utility פרסור ריכוזי `parseHeDate` ב-`app/lib/tenderMeta.ts` — מזהה `DD/MM/YYYY`
  ו-`DD.MM.YYYY` ומפרסר יום/חודש נכון; ISO נתמך כרגיל. `fmtDate` ו-`daysLeft`
  (דף הפרט) וכן `fd`/`dl` (הדשבורד) עברו כולם להשתמש בו — נקודת פרסור אחת.
- ה-API של דף הפרט (`/api/tender/[id]`) ממיר עכשיו את כל התאריכים מהמקור
  ל-ISO חד-משמעי לפני ההחזרה ללקוח.
- "תאריך פרסום" ריק: נוסף fallback שמאתר את התאריך לפי תווית בטקסט המלא
  של העמוד כשהחילוץ המבני מפספס.

## ✅ TICKET-12 — איחוד מנוע החיפוש והסינון לפי תחום
**סוג:** Bug/Improvement · **עדיפות:** גבוהה · **סטטוס: בוצע (יולי 2026)**

- נוספה פונקציית התאמה מרכזית `matchesQuery` בדשבורד: חיפוש חופשי בודק
  substring כרגיל, ובנוסף — אם השאילתה תואמת תחום (לפי שם התחום או מילה
  מהאשכול שלו), כל מכרז שפוגע באשכול נחשב תואם.
- תוצאה: חיפוש "טכנולוגיה" מחזיר את כל מכרזי אשכול הטכנולוגיה (כמו הסינון),
  ולא רק התאמות מילוליות. אשכולות המילים (BIZ) הם מקור אמת אחד לסינון ולחיפוש.
