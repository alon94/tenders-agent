# Contributing to tenders-agent

Thank you for your interest in contributing! This document outlines the process for contributing to this project.

## How to Contribute

1. **Fork** the repository
2. 2. **Create a branch** from `main` with a descriptive name (e.g. `feature/my-feature` or `fix/bug-description`)
   3. 3. **Make your changes** and commit them with clear messages
      4. 4. **Push** your branch to your fork
         5. 5. **Open a Pull Request** against the `main` branch
           
            6. ## Pull Request Guidelines
           
            7. - Keep PRs focused on a single feature or fix
               - - Write a clear description of what the PR does and why
                 - - Reference any related issues with `#issue-number`
                   - - Ensure your code is tested before submitting
                    
                     - ## Commit Message Format
                    
                     - Use clear, descriptive commit messages:
                     - ```
                       feat: add filtering by region
                       fix: correct deadline calculation
                       docs: update README with setup instructions
                       ```

                       ## Code Style

                       - Follow existing code conventions in the project
                       - - Add comments for complex logic
                        
                         - ## Questions?
                        
                         - Open an issue and we will get back to you.
